Очень целесообразно выбрать для последующего рефакторинга именно эту задачу:

 - Три класса вынуждены использовать Сканнер, в каждом из них он остается не закрытым.
Закрытие одного из них приводит к невозможностио остальных классов использовать свои сканнеры для своих целей.

 - Чтение/запись - классы, которые прописывались на ощупь. 
 Есть необходимость разобрать подробнее, скорее всего код можно упростить.
 В текущей реализации запись в файл осуществляется разом для всего пула данных,
 нет возможности дозаписывать файл при создании новых задач. Нужен отдельный метод именно для записи.

  - Подготовка данных для записи занимает большой объем, возможно, ее можно значительно упростить.

  - Можно дописать методы сортировки по остальным параметрам.

  Действительно удачной идеей выглядит только использование логгера.
